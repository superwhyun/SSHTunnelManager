# 개발 체크포인트 (Phase별 Go/No-Go)

> 목적: 각 Phase 완료 시 사용자 확인 후 다음 단계 진행  
> 형식: 체크리스트 + 결정사항

---

## 🚦 Phase 1: 기반 구축 (Checkpoint 1)

**완료 기준:** Electron 실행 → UI 표시 → 더미 데이터 저장/조회

### 확인 사항
- [ ] **UI 레이아웃** 만족하는가? (사이드바 + 상세 패널 구조)
- [ ] **데이터 구조** 적절한가? (Tunnel 필드 정의)
- [ ] **마스터 키 설정** UX 직관적인가? (첫 실행 시 암호 설정)

### 결정 필요 사항
1. **비밀번호 입력 방식**:
   - A) 메인 화면에서 바로 입력
   - B) 별도 다이얼로그 (Vault처럼)
   - C) 시스템 로그인 연동 (PIN/Biometric)

2. **데이터 저장 위치**:
   - A) 앱 폰더 내 SQLite (`~/.config/reverse-tunnel/data.db`)
   - B) 사용자 지정 위치 (NAS/Dropbox 동기화용)

**➡️ 사용자 승인 후 Phase 2 진행**

---

## 🚦 Phase 2: SSH 핵심 기능 (Checkpoint 2)

**완료 기준:** 실제 SSH 연결 → 터널 작동 → 로그 표시

### 확인 사항
- [ ] **인증 방식** 테스트 완료? (비밀번호/키 둘 다)
- [ ] **에러 메시지** 이해하기 쉬운가? (Permission denied → "비밀번호/키 확인")
- [ ] **로그 표시** 적절한가? (Real-time vs 버퍼링)

### 결정 필요 사항
1. **SSH 연결 실패 시**:
   - A) 무한 재시도 (사용자가 Stop 버튼 누를 때까지)
   - B) 3회 재시도 후 대기 (수동 재시작)
   - C) 점진적 백오프 (1분 → 5분 → 15분)

2. **보안 수준**:
   - A) 기본: 암호화된 SQLite에 저장
   - B) 높음: OS 키체인 통합 (macOS Keychain/Windows Credential)

3. **Known Hosts 처리**:
   - A) 첫 연결 시 자동 수락 (편의성 ↑)
   - B) 지문 확인 다이얼로그 (보안 ↑)

**➡️ 사용자 승인 후 Phase 3 진행**

---

## 🚦 Phase 3: 관리 기능 (Checkpoint 3)

**완료 기준:** 여러 터널 동시 관리 + 트레이 아이콘 + 기본 설정

### 확인 사항
- [ ] **동시 연결 5개 이상** 테스트? (성능 문제 없는지)
- [ ] **트레이 아이콘** UX 적절한가? (Windows/macOS/Linux)
- [ ] **단축키** 필요한가? (Ctrl+Shift+T 등)

### 결정 필요 사항
1. **트레이 메뉴 범위**:
   - A) 간단: 연결/해제 토글만
   - B) 풀기능: 새 터널 추가, 설정, 종료

2. **자동 시작 옵션**:
   - A) OS 부팅 시 모든 터널 자동 연결
   - B) "즐겨찾기" 지정된 터널만 자동 연결
   - C) 수동만 (보안 우선)

3. **앱 종료 시 동작**:
   - A) 모든 터널 끊고 종료
   - B) 백그라운드로 최소화 (트레이 유지)
   - C) 설정에 따라

**➡️ 사용자 승인 후 Phase 4 진행**

---

## 🚦 Phase 4: 고급 기능 (Checkpoint 4 - 최종)

**완료 기준:** 자동 재연결 + 통계 + 설정 백업

### 확인 사항
- [ ] **장기간 실행** 안정적인가? (24시간+ 메모리 누수 체크)
- [ ] **설정 백업/복원** 직관적인가?

### 결정 필요 사항
1. **통계 보관 기간**:
   - A) 7일
   - B) 30일
   - C) 무제한 (DB 용량 고려)

2. **업데이트 매커니즘**:
   - A) 자동 업데이트 (electron-updater)
   - B) 수동 다운로드 안내
   - C) 스토어 배포 (Mac App Store/Windows Store)

3. **기능 확장 (향후)**:
   - A) SOCKS5 프록시 지원 (`ssh -D`)
   - B) 포트 포워딩 규칙 템플릿
   - C) 팀 공용 설정 (클릭 한 번으로 동료 설정 가져오기)

---

## 🚨 핵심 결정사항 요약

Phase 1~4 중 **반드시 사용자가 결정해야 할 5가지**:

| # | 결정사항 | 기본값 | 영향 범위 |
|---|----------|--------|-----------|
| 1 | 마스터 키 UX 방식 | B) 별도 다이얼로그 | 보안/사용성 |
| 2 | 자동 재연결 정책 | C) 점진적 백오프 | 리소스/배터리 |
| 3 | Known Hosts 처리 | B) 지문 확인 | 보안 수준 |
| 4 | 앱 종료 시 동작 | C) 설정에 따라 | UX |
| 5 | 통계 보관 기간 | B) 30일 | 디스크 사용량 |

```
    ↓

```

## 💡 사용자 피드백 템플릿

각 체크포인트에서 사용자에게 물을 질문:

1. **"지금 화면/동작이 예상했던 것과 같은가?"** (예/아니오)
2. **"불편한 점이 있는가?"** (자유 입력)
3. **"추가로 필요한 기능이 있는가?"** (자유 입력)
4. **결정사항 선택** (A/B/C 중 택일)
